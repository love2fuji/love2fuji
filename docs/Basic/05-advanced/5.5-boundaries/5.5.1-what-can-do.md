# 5.5.1 Vibe Coding 能做什么

经过本节学习，你将了解：
- Vibe Coding 真正擅长的六大场景
- 2025 年 AI 编程的真实数据
- 为什么你走在正确的路上

## 先看数据：AI 编程已是主流

在讨论 Vibe Coding 能做什么之前，先看一组 2025 年的真实数据：

| 数据 | 来源 |
|------|------|
| **41%** 的代码现在由 AI 生成或辅助 | Stack Overflow 2025 开发者调查 |
| **84%** 的开发者正在使用或计划使用 AI 编程工具 | Index.dev 2025 报告 |
| **82%** 的开发者每周使用 AI 编程工具 | Second Talent 2025 统计 |
| **25%** 的新代码由 AI 辅助完成 | Google 2025 官方数据 |

这些数字说明：**你学习 Vibe Coding，不是在追赶潮流，而是在拥抱已经到来的现实。**

## Vibe Coding 擅长的六大场景

### 场景一：个人工具

**为什么擅长**：需求由你自己定义，不需要考虑复杂的用户权限、多人协作、海量数据。AI 能完整理解你的意图。

| 项目类型 | 复杂度 | 成功率 |
|---------|--------|--------|
| 待办清单 | 低 | 很高 |
| 习惯打卡器 | 低 | 很高 |
| 个人记账本 | 中 | 高 |
| 读书笔记管理 | 中 | 高 |
| 个人密码管理（本地版） | 中 | 高 |

**真实案例**：你在第四章做的待办清单，就是典型的个人工具。从零到能用，可能只花了一两个小时。

### 场景二：原型验证

**为什么擅长**：原型的目标是"快速验证想法是否可行"，不需要完美，不需要能承载百万用户。Vibe Coding 的速度优势在这里发挥得淋漓尽致。

根据 Index.dev 2025 的报告，使用 AI 编程工具的开发者：
- 完成原型的速度提升 **30%-75%**
- GitHub Copilot 用户每周完成的项目数量增加 **126%**

**典型场景**：
- 创业者验证一个产品想法
- 学生为课程项目做 Demo
- 向老板或客户演示一个概念

### 场景三：效率脚本

**为什么擅长**：脚本通常是"输入 → 处理 → 输出"的线性流程，逻辑清晰，没有复杂的状态管理。这正是 AI 最擅长的模式。

| 脚本类型 | 示例 |
|---------|------|
| 文件批处理 | 批量重命名、批量压缩图片、整理下载文件夹 |
| 数据转换 | Excel 转 JSON、CSV 合并、格式标准化 |
| 自动化任务 | 定时备份、自动发邮件提醒、爬取公开数据 |

**一个真实场景**：假设你每周要把 10 个 Excel 表格合并成一个汇总表。手动做需要 30 分钟，用 Vibe Coding 写个脚本可能只需要 10 分钟，而且以后每周只需要运行一下，几秒钟搞定。

### 场景四：数据分析与可视化

**为什么擅长**：数据分析有明确的输入（数据）和输出（图表/结论），中间的处理步骤可以清晰描述。AI 能很好地理解"我想看 XX 数据的 YY 趋势"这类需求。

| 分析类型 | 适合程度 |
|---------|---------|
| 销售数据趋势分析 | 很适合 |
| 问卷调查结果可视化 | 很适合 |
| 个人消费分析 | 很适合 |
| 简单的统计报告 | 很适合 |

### 场景五：静态网站

**为什么擅长**：静态网站没有复杂的后端逻辑，主要是"展示内容"。HTML + CSS + 少量 JavaScript，是 AI 最熟练的领域之一。

| 网站类型 | 复杂度 | 成功率 |
|---------|--------|--------|
| 个人简历页面 | 低 | 很高 |
| 作品集展示 | 低 | 很高 |
| 活动宣传页 | 低 | 很高 |
| 个人博客（静态） | 中 | 高 |
| 小型企业官网 | 中 | 高 |

### 场景六：学习编程概念

**为什么擅长**：AI 可以用你能理解的语言解释代码，帮你建立直觉。你不需要先学会语法才能开始做东西，而是在做的过程中逐渐理解。

这正是 Vibe Coding 的核心价值：**降低入门门槛，让"做中学"成为可能。**

## 一个共同特征：问题边界清晰

如果你仔细看上面六个场景，会发现它们有一个共同特征：

> **问题的边界是清晰的。**

- 个人工具：你自己用，需求你说了算
- 原型验证：只需要"能演示"，不需要"能上线"
- 效率脚本：输入输出明确
- 数据分析：数据是固定的，分析目标是明确的
- 静态网站：内容是确定的，交互很简单
- 学习概念：目标是"理解"，不是"生产"

当问题边界清晰时，AI 能很好地理解你的意图，生成的代码大概率能直接用。

## 本节要点

- 2025 年，41% 的代码由 AI 生成或辅助，AI 编程已是主流
- Vibe Coding 擅长六大场景：个人工具、原型验证、效率脚本、数据分析、静态网站、学习概念
- 这些场景的共同特征是"问题边界清晰"
- 在这些领域，大胆尝试，快速出成果
