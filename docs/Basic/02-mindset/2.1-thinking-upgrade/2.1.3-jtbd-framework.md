# 2.1.3 JTBD框架应用：四个场景实战

学会了JTBD的理论，现在来看看它如何应用到你实际想做的事情上。

我们会通过四个不同场景的示例，展示如何用任务视角重新定义问题。


## 任务描述模板

在开始之前，先记住这个模板。后面所有的分析都会用到它：

```
当 [某类用户] 在 [某个情境] 下，
他们想要 [完成某个任务]，
以便于 [获得某种结果/感受]。

目前他们的替代方案是 [现有解决方式]，
但这个方案的问题是 [痛点]。
```


## 场景一：产品原型——待办清单

### 原始想法
"我想做一个待办清单App"

### 用JTBD重新分析

**用户是谁？**
小李，互联网公司运营，每天要处理20-30个工作事项

**情境是什么？**
每天早上到公司，需要理清今天要做什么

**要完成什么任务？**
快速记录和查看今天要做的事，确保不遗漏

**期望的结果是什么？**
安心投入工作，不用担心漏掉重要的事

**目前的替代方案？**
用便利贴、微信置顶消息、或脑子记

**替代方案的问题？**
便利贴容易丢，微信消息会被刷掉，脑子记不住

### JTBD视角的任务描述

```
当小李每天早上到公司时，
他想要快速记录和查看今天要做的事，
以便于不遗漏重要任务，可以安心投入工作。

目前他用便利贴和微信置顶来管理，
但便利贴容易丢，微信消息会被别的聊天刷掉。
```

### 这个分析改变了什么？

有了这个分析，你和AI的对话会完全不同：

**之前（功能视角）：**
> "帮我做一个待办清单，要有分类、标签、日历视图……"

**之后（任务视角）：**
> "帮我做一个简单的每日待办工具。用户是职场人士，早上需要快速记录今天要做的事，核心需求是'不遗漏'和'快速记录'。不需要复杂功能，能添加任务、标记完成就够了。"


## 场景二：数据分析——Q3销售报告

### 原始想法
"我想做一个销售数据可视化看板"

### 用JTBD重新分析

**用户是谁？**
销售部门负责人，需要向管理层汇报季度业绩

**情境是什么？**
下周要开季度会议，需要准备汇报材料

**要完成什么任务？**
清楚展示Q3销售表现，回答"我们做得怎么样？"

**期望的结果是什么？**
让管理层快速理解业绩情况，为Q4策略提供依据

**目前的替代方案？**
手动在Excel里做图表，再复制到PPT

**替代方案的问题？**
每次都要重新做，格式不统一，耗时2-3小时

### JTBD视角的任务描述

```
当销售负责人准备季度汇报时，
他想要清楚展示Q3的销售表现，
以便于让管理层快速理解业绩情况，做出Q4决策。

目前他用Excel做图表再复制到PPT，
但每次都要重新做，格式不统一，耗时长。
```

### 这个分析改变了什么？

**之前（功能视角）：**
> "帮我做一个销售看板，展示趋势、留存、转化……"

**之后（任务视角）：**
> "帮我做一个Q3销售汇报页面。受众是管理层，他们关心的核心问题是'Q3做得怎么样'和'Q4应该怎么调整'。数据我会提供Excel，需要展示总销售额、月度趋势、和各区域对比。"


## 场景三：自动化脚本——Excel汇总

### 原始想法
"我想写一个Python脚本处理Excel"

### 用JTBD重新分析

**用户是谁？**
行政助理，负责汇总各部门周报

**情境是什么？**
每周五下午，需要把5个部门的周报汇总成一份总表

**要完成什么任务？**
把5个Excel合并成1个，计算汇总数据

**期望的结果是什么？**
下班前完成汇总，不用加班

**目前的替代方案？**
手动复制粘贴每个部门的数据

**替代方案的问题？**
容易出错，格式不统一，要花2小时

### JTBD视角的任务描述

```
当行政助理每周五下午需要汇总周报时，
她想要快速把5个部门的Excel合并成一份总表，
以便于准时下班，不用加班。

目前她手动复制粘贴每个部门的数据，
但容易出错、格式不统一，每次要花2小时。
```

### 这个分析改变了什么？

**之前（技术视角）：**
> "帮我写Python脚本，用Pandas读取Excel，合并数据……"

**之后（任务视角）：**
> "帮我自动化一个周报汇总流程。每周我会收到5个Excel文件，格式相同。需要把它们合并成一份总表，并计算合计行。输出一个新的Excel文件。"


## 场景四：给家人做工具——吃药提醒

### 原始想法
"我想给我爸做一个吃药提醒App"

### 用JTBD重新分析

**用户是谁？**
60岁的父亲，每天需要吃3种药

**情境是什么？**
每天早晚饭后需要吃药，经常忘记

**要完成什么任务？**
按时吃药，不漏服

**期望的结果是什么？**
身体健康，子女放心

**目前的替代方案？**
让家人提醒，或者在药盒上贴纸条

**替代方案的问题？**
家人不总在身边，纸条容易忽略

### JTBD视角的任务描述

```
当60岁的父亲每天需要吃药时，
他想要有东西提醒他按时服药，
以便于保持健康，让子女放心。

目前他靠家人提醒或药盒上的纸条，
但家人不总在身边，纸条容易被忽略。
```

### 这个分析改变了什么？

**之前（功能视角）：**
> "帮我做一个吃药提醒App，要有药品管理、用量记录、库存追踪……"

**之后（任务视角）：**
> "帮我给60岁的父亲做一个吃药提醒网页。他不太会用手机App，所以要极简设计：大字体、醒目颜色、每天早晚两个时间点提醒。只需要响铃提醒和'已服药'按钮，不需要其他功能。"


## 传统思维 vs JTBD思维对比

| 维度 | 传统思维 | JTBD思维 |
|-----|---------|---------|
| 核心问题 | "我要做什么功能？" | "用户要完成什么任务？" |
| 起点 | 功能清单 | 用户场景 |
| 关注点 | 产品有什么 | 用户需要什么 |
| 竞争分析 | 别人有什么功能 | 用户现在怎么解决 |
| 成功标准 | 功能做完了 | 任务完成得更好 |


## 识别"真正的任务"三步法

当你有了一个想法，用这三步来识别真正的任务：

### 第一步：问"为什么"

> 我想做一个XXX。
> → 为什么？

继续问"为什么"，直到你找到真正的动机。

**示例：**
- 我想做一个习惯打卡App → 为什么？
- 因为我想养成跑步习惯 → 为什么？
- 因为我想减肥 → 为什么？
- 因为我想更健康、更自信

真正的任务是"变得更健康更自信"，不是"做一个App"。

### 第二步：问"现在怎么做"

> 你现在怎么解决这个问题？

了解现有方案，能帮你理解真正的痛点。

### 第三步：问"哪里不爽"

> 现在的方案有什么问题？

痛点往往藏在这里。你的解决方案只需要比现有方案好一点点，就能被"雇佣"。
