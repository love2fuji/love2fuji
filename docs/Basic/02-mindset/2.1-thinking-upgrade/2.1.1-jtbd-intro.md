# 2.1.1 JTBD框架：理解"雇佣"隐喻

## 什么是JTBD

Jobs to be Done（待完成的任务），简称JTBD，是由哈佛商学院教授Clayton Christensen提出的产品思维框架。

**核心思想只有一句话：**

> 人们不是在购买产品本身，而是在"雇佣"产品来帮助他们完成某项任务。

"雇佣"这个词用得很妙。想象一下：

- 你雇佣一个清洁工，是为了让家里变干净
- 你雇佣一个会计，是为了让账目清晰
- 你雇佣一个待办清单App，是为了不遗漏重要的事

如果清洁工打扫得不干净，你会解雇他。如果App没能帮你记住事情，你也会卸载它。

**产品被"雇佣"是因为它能完成任务，被"解雇"是因为它完成得不好。**


## 奶昔的故事：JTBD的经典案例

这是JTBD框架最著名的案例，来自一家快餐连锁店的真实经历。

### 问题

这家连锁店想要提高奶昔的销量。他们最初的做法是：

- 调查顾客："你想要奶昔更甜还是更稠？"
- 增加口味选择：草莓、巧克力、香草……
- 优化配料组合

结果呢？销量几乎没有变化。

### 转折

后来，研究团队换了一个问题：

> "顾客在什么情况下会'雇佣'奶昔？他们要完成什么任务？"

他们派人去店里观察，发现了一个有趣的现象：

**大多数奶昔是在早上7点到9点之间卖出的，买奶昔的人通常是独自开车上班的通勤者。**

### 洞察

通过访谈，团队发现这些顾客"雇佣"奶昔的真正任务是：

| 顾客的真实任务 | 为什么选择奶昔 |
|--------------|---------------|
| 在漫长的通勤路上有事可做 | 吸管喝奶昔可以单手操作，不影响开车 |
| 抵御上午的饥饿感 | 奶昔比咖啡更顶饿，能撑到午饭 |
| 比面包不那么无聊 | 有味道，吃起来不枯燥 |
| 保持衣服整洁 | 不会像三明治那样掉渣 |

### 解决方案

基于这个洞察，团队的建议是：**让奶昔更稠**。

更稠的奶昔需要更长时间才能吸完，这意味着：
- 通勤路上的"陪伴"时间更长
- 顾客会觉得更"值"

结果：奶昔销量提升了超过40%。

### 这个故事告诉我们什么

传统思维问的是"你想要什么口味"，这是**功能视角**。

JTBD思维问的是"你要完成什么任务"，这是**任务视角**。

同样是卖奶昔，视角不同，答案完全不同。


## 任务的三个层次

JTBD框架把用户的任务分为三个层次：

| 层次 | 定义 | 示例（待办清单场景） |
|-----|------|-------------------|
| **功能任务** | 用户要完成的具体事情 | 记录今天要做的事、设置截止日期 |
| **情感任务** | 用户想要的感受 | 感到安心、有掌控感、不焦虑 |
| **社会任务** | 用户想要的社会形象 | 显得有条理、专业、靠谱 |

**初学者常犯的错误是只看到功能任务，忽略情感任务和社会任务。**

举个例子：

一个职场新人下载待办清单App，表面上是为了"记录任务"（功能任务），但深层原因可能是：
- 害怕遗漏重要事项导致被领导批评（情感任务）
- 想让同事觉得自己是个靠谱的人（社会任务）

如果你只关注功能，可能会不断加功能：分类、标签、统计……

但如果你理解情感任务，可能会发现：一个简单的"今日待办"清单，加上完成时的小庆祝动画，反而更能解决问题。


## 为什么JTBD对Vibe Coding特别重要

你可能会问：这个框架是给产品经理用的，和我用AI写代码有什么关系？

关系太大了。

**AI是一个超级执行者，但它不会帮你思考"该做什么"。**

当你告诉AI"帮我做一个待办清单"，AI会忠实地执行你的指令。但它不会问你：

- 你为什么需要待办清单？
- 你现在用什么方式管理任务？
- 那个方式有什么问题？

这些问题，只有你自己能回答。

**JTBD框架的价值，就是帮你在开口告诉AI之前，先想清楚这些问题。**


## 一个思维转换练习

下次当你有了一个想法，试着完成这个转换：

| 原始想法 | 问自己 | JTBD视角的重述 |
|---------|--------|---------------|
| 我想做一个待办清单 | 我为什么需要它？ | 当我每天开始工作时，我想快速记录今天要做的事，这样我就不会遗漏重要任务，可以安心投入工作 |
| 我想做一个记账App | 我为什么需要它？ | 当我月底发现钱不知道花哪了，我想看清楚钱都去了哪里，这样我可以调整消费习惯，存下更多钱 |
| 我想写一个自动化脚本 | 我为什么需要它？ | 当我每周都要手动汇总5个部门的Excel表格，我想让这个过程自动完成，这样我可以把时间花在更有价值的事情上 |

注意这个转换的结构：

```
当 [某个情境发生时]，
我想要 [完成某个任务]，
这样我就可以 [获得某种结果/感受]。
```

这个结构会在后面的练习中反复使用。先记住它。
