# 2.7.5 给不同人群的问题发现建议

每个人的生活场景不同，发现问题的方式也不同。本节为四类典型读者提供针对性的问题发现指南。


## 如果你是职场人士

你的优势：**工作流程中充满了可优化的环节**。

### 三个高频问题方向

| 方向 | 典型问题 | MVP 形态 |
|-----|---------|---------|
| **数据汇总** | 每周从多个来源收集数据做报表 | 自动汇总脚本 |
| **信息同步** | 需要在多个系统之间手动复制信息 | 数据搬运工具 |
| **重复沟通** | 反复回答相同的问题或发送相同的内容 | 常用回复模板库 |

### 从问题到项目的完整路径

**场景**：你是一名市场运营，每周要做一份渠道数据周报。

**烦恼**：需要从 5 个不同的广告平台导出数据，复制到 Excel，手动计算汇总指标，然后做成 PPT。整个过程要花 2-3 小时。

**问题拆解**：
1. 数据导出：目前需要手动登录 5 个平台 → 暂时无法自动化（涉及账号安全）
2. 数据汇总：把 5 个 Excel 合并成一个 → 可以自动化
3. 指标计算：计算环比、同比、转化率 → 可以自动化
4. 生成报告：做成可视化图表 → 可以自动化

**最小方案**：做一个工具，能自动合并 5 个 Excel 文件，计算核心指标，输出汇总表格。

**五维评分**：
- 重复性：每周一次 → 4 分
- 规则性：列名固定，计算公式固定 → 5 分
- 可验证性：可以对照原数据验算 → 5 分
- 敏感性：公司内部数据，中等敏感 → 3 分
- 容错性：做错了可以重新跑 → 4 分
- **总分：21 分**，非常适合

### 职场人士的发现技巧

**观察你的工作日志**：连续三天，记录你做的每一件事和花费的时间。然后问自己：
- 哪些事情花的时间和产出不成比例？
- 哪些事情是"搬砖"性质的？
- 哪些事情你做的时候脑子可以放空？

这些"脑子可以放空"的任务，往往就是可以自动化的任务。


## 如果你是学生

你的优势：**学习场景中有大量信息管理和时间管理的需求**。

### 三个高频问题方向

| 方向 | 典型问题 | MVP 形态 |
|-----|---------|---------|
| **DDL 管理** | 作业、考试、活动的截止日期散落各处 | DDL 聚合提醒工具 |
| **笔记整理** | 课堂笔记、读书笔记、资料收集无法有效回顾 | 简易笔记索引系统 |
| **学习追踪** | 不知道自己学了多少、进度如何 | 学习打卡/进度追踪器 |

### 从问题到项目的完整路径

**场景**：你是一名大三学生，同时在准备考研、实习和课程作业。

**烦恼**：DDL 来自微信群、邮件、教务系统、实习公司的钉钉……经常漏掉重要截止日期。

**问题拆解**：
1. DDL 来源分散 → 需要一个统一的收集入口
2. 没有提醒机制 → 需要在截止日期前提醒
3. 无法一眼看清优先级 → 需要按时间排序

**最小方案**：做一个简单的 DDL 收集页面。手动添加每个 DDL（来源、截止时间、重要程度），按时间排序显示，临近截止时高亮提醒。

**五维评分**：
- 重复性：每天都会看 → 5 分
- 规则性：添加和显示规则固定 → 5 分
- 可验证性：DDL 到没到一眼能看出来 → 5 分
- 敏感性：只是自己的日程 → 5 分
- 容错性：漏了一个重新添加就行 → 5 分
- **总分：25 分**，完美适合

### 学生的发现技巧

**回忆你的"焦虑时刻"**：
- 什么时候你会感到焦虑？（考试前、DDL 前、选课时）
- 焦虑的根源是什么？（信息不清楚、进度不可控、资料找不到）
- 如果有一个工具能帮你，它应该做什么？

焦虑往往指向真实的问题。


## 如果你是自由职业者/创业者

你的优势：**业务流程完全由自己控制，可以自由优化**。

### 三个高频问题方向

| 方向 | 典型问题 | MVP 形态 |
|-----|---------|---------|
| **客户管理** | 客户信息散落在微信、邮件、笔记里 | 简易客户名录 |
| **报价/合同** | 每次都要改模板，容易出错 | 报价单生成器 |
| **项目追踪** | 多个项目并行，不知道哪个该催款了 | 项目状态看板 |

### 从问题到项目的完整路径

**场景**：你是一名自由设计师，同时服务 8-10 个客户。

**烦恼**：每个客户的项目进度、付款状态、下次跟进时间都在脑子里记着，经常忘记催尾款。

**问题拆解**：
1. 项目信息没有统一记录 → 需要一个项目列表
2. 状态变化不好追踪 → 需要状态标签（进行中/待付款/已完成）
3. 不知道什么时候该跟进 → 需要下次跟进日期提醒

**最小方案**：做一个项目看板。每个项目卡片包含：客户名、项目内容、当前状态、尾款金额、下次跟进日期。按状态分列显示。

**五维评分**：
- 重复性：每天都要看 → 5 分
- 规则性：状态流转固定 → 5 分
- 可验证性：项目有没有跟进一目了然 → 5 分
- 敏感性：涉及客户信息和金额，中等敏感 → 3 分
- 容错性：记错了可以改 → 4 分
- **总分：22 分**，非常适合

### 自由职业者的发现技巧

**复盘你的"漏单时刻"**：
- 什么时候你忘了跟进客户？
- 什么时候你报价报错了？
- 什么时候你发票开晚了？

这些"漏单"背后，往往藏着系统性的问题。


## 如果你想给家人做工具

你的优势：**你最了解家人的需求，也最有动力帮他们解决问题**。

### 三个高频问题方向

| 方向 | 典型问题 | MVP 形态 |
|-----|---------|---------|
| **操作简化** | 手机功能太多，找不到常用的 | 大字版快捷入口 |
| **定时提醒** | 吃药、浇花、缴费容易忘 | 语音/短信提醒工具 |
| **信息聚合** | 想看天气、快递、新闻，但不会操作 | 一页式信息看板 |

### 从问题到项目的完整路径

**场景**：你妈妈 62 岁，智能手机用了 5 年，但还是经常打电话问你"怎么看天气""怎么给你爸打电话"。

**烦恼**：手机主屏上有 30 多个 App，她常用的其实只有 5 个，但每次都要找半天。

**问题拆解**：
1. 入口太多 → 需要简化到只有常用功能
2. 图标太小 → 需要放大
3. 文字看不清 → 需要大字体
4. 不知道点哪个 → 需要明确的文字标签

**最小方案**：做一个网页，放 5 个超大按钮：打电话给儿子、打电话给老伴、看天气、看微信、看照片。每个按钮配上清晰的文字标签，设为浏览器主页。

**五维评分**：
- 重复性：每天都用 → 5 分
- 规则性：点击→跳转，完全固定 → 5 分
- 可验证性：能不能用一试就知道 → 5 分
- 敏感性：不涉及敏感信息 → 5 分
- 容错性：点错了退出重来就行 → 5 分
- **总分：25 分**，完美适合

### 给家人做工具的发现技巧

**观察而非询问**：
- 不要问"你需要什么工具"（他们不知道）
- 要观察"他们做什么事情时卡住了"

**记录他们的"求助"**：
- 每次家人打电话问你怎么操作，记下来
- 问的次数最多的问题，就是最值得解决的问题

**设计时谨记**：
- 按钮要大（至少占屏幕宽度 40%）
- 文字要大（至少 24px）
- 步骤要少（最好一步到位）
- 颜色要对比强烈（深色背景配浅色文字，或反过来）


## 如果你完全没思路

有些人确实想不出任何问题。这很正常。两个建议：

### 方法一：从模仿开始

去 GitHub、Product Hunt、少数派等平台，看看别人做了什么小工具。挑一个你觉得"这个我也需要"的，复刻一个自己的版本。

在复刻的过程中，你会发现：
- 原版哪里不好用 → 改进点
- 你还需要什么功能 → 新需求
- 这个工具可以用在其他场景 → 新想法

### 方法二：从请求开始

问你身边的人："有什么事情你希望有人帮你做？"

- 问你的同事："你平时做什么事情最烦？"
- 问你的家人："你最近用手机遇到什么困难？"
- 问你的朋友："你有没有什么重复的任务特别想自动化？"

别人的问题，可能就是你的项目。


## 核心要点

✓ 不同人群的问题发现渠道不同，要从自己的生活场景出发

✓ 职场人士关注工作流程，学生关注信息管理，自由职业者关注客户管理

✓ 给家人做工具时，观察比询问更有效

✓ 完全没思路时，可以从模仿开始，或者直接问身边的人
