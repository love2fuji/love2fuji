# 2.7.7 本节核心要点

## 三条核心原则

✓ **问题不是想出来的，是观察出来的。** 每一个"好烦"的瞬间，都是发现问题的线索。通过五种渠道（重复劳动、身边人抱怨、现有工具不满、工作流断点、信息获取困难），系统性地扫描你的日常生活。

✓ **用"烦恼日记"捕捉问题，用五维评分筛选问题。** 连续 7 天记录烦恼，然后用重复性、规则性、可验证性、敏感性、容错性五个维度打分，选出最适合用 Vibe Coding 解决的问题。

✓ **知道什么不做，和知道什么要做一样重要。** 涉及资金、隐私、监管、高并发、长期维护的问题，不适合用 Vibe Coding 解决。识别边界，避免踩坑。


## 问题发现检查清单

在动手做项目之前，确认你已经完成以下步骤：

```markdown
## 问题发现自检清单

### 问题来源
- [ ] 我已经识别出这个问题来自哪个渠道（重复劳动/他人抱怨/工具不满/流程断点/信息困难）
- [ ] 我记录了这个问题发生的频率和痛苦程度

### 问题筛选
- [ ] 我已经用五维评分法评估了这个问题
- [ ] 总分达到 15 分以上
- [ ] 这个问题不在"红线清单"内（不涉及资金/隐私/监管）

### 问题定义
- [ ] 我能用一句话描述"谁有这个问题"
- [ ] 我能用一句话描述"痛点具体是什么"
- [ ] 我能用一句话描述"为什么现有方案解决不了"

### MVP 描述
- [ ] 我已经写出了 MVP 的一句话描述
- [ ] 这个描述足够具体，AI 能理解我要做什么
```

## 常见问题

**Q：我做完练习，发现我的问题评分很低，怎么办？**

A：这是好事！说明你避免了一个不适合的问题。回到烦恼日记，换一个问题评估。也可以考虑把问题"降级"到更安全的范围（比如只做个人使用的版本）。

**Q：我的问题只有我自己有，这样值得做吗？**

A：完全值得。给自己做一个好用的工具，本身就是 Vibe Coding 的价值所在。而且，解决自己问题的过程中，你会学到技能、建立信心，为将来做更大的项目打下基础。

**Q：我做完访谈，发现别人不太有这个问题，怎么办？**

A：两种选择：(1) 把它当作个人工具来做，不追求普遍性；(2) 重新理解问题，也许你问的方式需要调整，也许问题的本质和你想的不一样。
