# 2.7.2 问题就在你身边，只是你没注意

## 一个思维实验

回想一下过去一周，你有没有：

- 做过重复的手工操作？
- 在多个地方复制粘贴同样的信息？
- 等待某个人给你发送某个文件？
- 用 Excel 做过让你觉得"好烦"的事？
- 想找某个信息但找不到？
- 觉得某个 App 不好用但又不得不用？

**每一个"好烦"的瞬间，都是一个潜在的问题。**

问题是：大多数人在"烦"完之后就忘了。他们继续忍受，继续抱怨，但从不停下来想一想——"这个问题，能不能解决？"


## 五种发现问题的渠道

问题不会主动跳出来告诉你"我在这里"。你需要知道去哪里找。

以下是五种最有效的问题发现渠道：

### 渠道一：自己的重复劳动

**观察点**：什么事情你每天/每周都在做，而且做的时候感觉"又来了"？

| 信号 | 示例 |
|-----|------|
| "每次都要……" | 每次开会前都要手动整理会议纪要模板 |
| "又要……" | 又要从5个Excel里汇总数据做周报 |
| "总是……" | 总是忘记给家人发提醒 |

**真实案例：Timesheetr 的诞生**

一位独立开发者厌倦了传统的时间表系统：每次记录工作时间都要打开复杂的软件，填写繁琐的表单，然后导出数据提交给 ERP 系统。他问自己："有没有可能做一个极简的工具，打开就能记，记完自动同步？"

于是他用几周时间做了一个轻量级的时间追踪工具 Timesheetr。核心功能只有三个：快速记录任务、自动计算时长、一键同步到 ERP。这个工具解决了他自己的痛点，也帮助了成千上万有同样困扰的自由职业者和咨询师。

**问自己**：过去一周，你重复做了什么事情超过 3 次？


### 渠道二：身边人的抱怨

**观察点**：家人、朋友、同事在抱怨什么？

| 信号 | 示例 |
|-----|------|
| "我妈说……" | 我妈说手机上的图标太多了，找不到想用的 |
| "同事吐槽……" | 同事吐槽每次找历史文件都要翻半天 |
| "朋友问我……" | 朋友问我有没有什么工具可以记录读书笔记 |

**关键技巧：听"过去的行为"，而非"未来的意愿"**

这是《The Mom Test》一书的核心洞见：当你问别人"你觉得这个工具怎么样"或"你会用吗"，得到的答案往往是礼貌性的肯定。但如果你问"你上次遇到这个问题是什么时候？你当时怎么解决的？"，你会得到真实的信息。

| 不要问 | 要问 |
|-------|------|
| "你觉得这个想法好不好？" | "你上次遇到这个问题是什么时候？" |
| "你会用这样的工具吗？" | "你现在怎么解决这个问题？花了多少时间？" |
| "这个功能有用吗？" | "你之前试过什么方法？为什么不管用？" |


### 渠道三：现有工具的不满

**观察点**：什么 App 或网站你在用，但用得很不爽？

| 信号 | 示例 |
|-----|------|
| "如果它能……就好了" | 如果这个日历能自动同步多个账户就好了 |
| "为什么它不能……" | 为什么这个笔记软件不能一键导出 PDF |
| "它什么都好，就是……" | 它什么都好，就是每次打开太慢了 |

**真实案例：Tag Timelines 的诞生**

一位父亲在使用 Apple AirTag 追踪孩子的书包和家里的狗时，发现了一个问题：AirTag 只能显示当前位置，不能查看历史轨迹。他想知道"孩子放学后去了哪里"，但苹果的 Find My 应用做不到。

他没有抱怨完就算了，而是问自己："我能不能自己做一个？"

于是他开发了 Tag Timelines，一个可以记录和展示 AirTag 位置历史的应用。这个工具不仅解决了他自己的问题，还吸引了大量有同样需求的用户——家长想追踪孩子的行踪，宠物主人想看狗狗在院子里跑了多远，商家想监控货物的运输路线。

**问自己**：你最近用什么工具时，心里嘀咕过"要是能……就好了"？


### 渠道四：工作流程的断点

**观察点**：在完成一件事的过程中，哪里"卡"了？

| 信号 | 示例 |
|-----|------|
| "我得先……然后……再……" | 我得先从系统A导出，然后手动整理，再导入系统B |
| "每次到这一步就卡住" | 每次到审批环节就要等三天 |
| "如果这两个能连起来……" | 如果微信群消息能自动同步到待办清单就好了 |

这种"断点"往往是最有价值的问题。因为它不是某个工具不好用，而是**两个工具之间没有打通**。

**常见的工作流断点**：
- 从会议录音到会议纪要（需要手动转录和整理）
- 从 Excel 数据到可视化图表（需要手动复制粘贴到另一个工具）
- 从想法到待办事项（需要打开另一个 App 手动输入）
- 从收据照片到报销单（需要手动填写信息）


### 渠道五：信息获取的困难

**观察点**：什么信息你经常需要，但每次获取都很麻烦？

| 信号 | 示例 |
|-----|------|
| "我得去哪里查……" | 我得去三个网站分别查比价 |
| "这个信息存在哪来着" | 这个客户的联系方式存在哪来着 |
| "每次都要问别人" | 每次都要问同事这个流程怎么走 |

**问自己**：什么信息你每周至少要找一次，每次都要花超过 2 分钟？


## 问题观察清单

用这份清单扫描你的日常生活，找出潜藏的问题：

```markdown
## 问题观察清单

### 重复劳动
- [ ] 我每天重复做的事情：________________
- [ ] 我每周重复做的事情：________________
- [ ] 做这些事时，最烦的环节是：________________

### 身边人的抱怨
- [ ] 家人最近抱怨过什么？________________
- [ ] 同事/朋友吐槽过什么工具？________________
- [ ] 有人向我求助过什么问题？________________

### 现有工具的不满
- [ ] 哪个常用 App 让我觉得"要是能……就好了"？________________
- [ ] 哪个网站/软件我用得很不顺手？________________

### 工作流断点
- [ ] 完成某件事时，哪两个步骤之间需要"手动搬运"数据？________________
- [ ] 哪些操作我总觉得"应该能自动化"？________________

### 信息获取困难
- [ ] 什么信息我经常要找，但总是记不住在哪？________________
- [ ] 什么数据我需要从多个地方拼凑？________________
```


## 一个重要提醒

发现问题只是第一步。不是所有问题都值得解决，也不是所有问题都适合用 Vibe Coding 解决。

下一节，我们会学习如何用"烦恼日记"系统性地记录问题，然后在 2.7.4 节学习如何筛选出真正值得投入的问题。

现在，先完成上面的观察清单。给自己 10 分钟，静下心来扫描一下你的日常生活。

你可能会惊讶地发现：问题一直都在，只是你之前没注意。
