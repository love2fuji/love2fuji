| 能力 | 具体表现 | 对应章节 |
|-----|---------|---------|
| **发现问题** | 能从日常生活中识别值得解决的问题 | 2.7 |
| **定义任务** | 能用"任务视角"描述用户需求，而不是列功能清单 | 2.1 |

这些能力的共同点是：**帮你想清楚"做什么"和"为什么做"**。


## 心法与技法的关系

本章叫"心法篇"，下一章叫"技法篇"。它们解决的是不同的问题：

| 维度 | 心法（第2章） | 技法（第3章） |
|-----|-------------|-------------|
| **核心问题** | 做什么？为什么做？ | 怎么让AI理解我要做什么？ |
| **输入** | 模糊的想法 | 清晰的需求 |
| **输出** | 清晰的需求 | AI能执行的指令 |
| **关键能力** | 产品思维、用户洞察 | Prompt工程、需求表达 |

**心法是技法的前提。**

如果你自己都没想清楚要做什么，写出来的Prompt也会是模糊的。AI会给你一个"看起来像那么回事"的结果，但不是你真正想要的。

反过来，如果你想得很清楚，但不知道怎么表达给AI，AI也无法帮你。

两者缺一不可。

## 一个预告：本章思考 → 下章Prompt

看看本章的输出如何变成下一章的输入：

**本章你完成的思考：**

```
用户：25岁职场新人，怕遗漏任务被批评
任务：快速记录今天要做的事，不遗漏重要任务
场景：早上到公司，打开电脑
MVP：添加任务、完成任务、查看今日任务
不做：分类、标签、提醒、统计
```

**下一章你将写出的Prompt：**

```markdown
## 背景
我是一个25岁的职场新人，经常因为忘记任务被领导批评。
我需要一个极简的每日待办工具。

## 用户场景
每天早上到公司，打开电脑，快速记下今天要做的事。
下班前检查是否都完成了。

## 核心功能（P0）
1. 添加任务：输入框 + 回车即可添加
2. 完成任务：点击任务左侧的圆圈打勾
3. 查看任务：打开页面就能看到今日所有任务

## 这一版不需要
- 任务分类或标签
- 截止日期或提醒
- 历史记录或统计
- 用户登录

## 设计要求
- 界面极简，打开就是任务列表
- 添加任务必须在3秒内完成
- 手机也能正常使用
```

**看到区别了吗？**

同样是"做一个待办清单"，经过本章的思考，你的Prompt从一句模糊的想法，变成了一份AI能精确执行的需求说明。


## 写在最后

如果你认真完成了这一章的学习和练习，恭喜你——你已经比大多数Vibe Coding初学者领先了一大步。

大多数人的做法是：有了想法 → 直接告诉AI → 发现不对 → 不断修改Prompt → 越改越乱 → 放弃。

你的做法将是：有了想法 → 想清楚 → 写出清晰的Prompt → AI一次给出接近预期的结果 → 微调 → 完成。

**区别不在于你用了什么AI工具，而在于你在开口之前想了多少。**


> **记住**
> 
> AI是一个超级执行者，但"做什么"只有你能决定。
> 
> 想清楚再动手，永远是最快的路。
