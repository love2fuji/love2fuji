# 2.2.6 本节核心要点

## 核心概念回顾

| 概念 | 定义 | 核心问题 |
|-----|------|---------|
| 逆向思维 | 通过思考"什么会导致失败"来规避风险 | 什么会让这个项目失败？ |
| Pre-mortem | 假设项目已经失败，追溯可能的原因 | 如果失败了，是因为什么？ |
| 避坑清单 | 把失败原因转化为预防措施 | 如何避免这些失败？ |


## 核心要点

✓ **避免失败比追求成功更容易**：你可能不知道成功的秘诀，但你通常能识别出明显的错误。先排除错误，再追求正确。

✓ **Pre-mortem：假设已经失败，追溯原因**：这个技术利用人类"后见之明"的能力，把"预测"变成"回顾"，可以将风险识别能力提高 30%。

✓ **把失败原因转化为预防措施**：识别风险只是第一步，关键是把每个高优先级的失败原因，转化为具体的预防措施，并落实到需求中。


## 本节检查清单

在开始你的项目之前，确保你能回答以下问题：

- [ ] 我做过 Pre-mortem 分析吗？
- [ ] 我列出了至少 5 个可能的失败原因吗？
- [ ] 我识别出了高可能性、高严重性的风险吗？
- [ ] 我为这些风险制定了具体的预防措施吗？
- [ ] 我把预防措施落实到需求描述中了吗？
