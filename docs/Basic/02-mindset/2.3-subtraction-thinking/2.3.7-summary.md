# 2.3.7 本节核心要点


## 三条核心原则

```
✓ MVP 是实验，不是产品。它的目的是验证假设，而不是取悦用户。

✓ 做减法比做加法更难，但更重要。功能多不等于价值高。

✓ 「不做清单」和「要做清单」同样重要。明确边界才能聚焦核心。
```


## 本节思维模型回顾

| 概念 | 核心要点 | 应用场景 |
|-----|---------|---------|
| MVP 重定义 | Minimum = 最小投入，Viable = 能验证假设 | 任何需要决定「做什么」的时候 |
| 三个灵魂问题 | 能用吗？验证假设吗？首次需要吗？ | 筛选功能优先级 |
| P0/P1/P2 | P0 ≤ 5 个，其他都可以暂缓 | 确定 MVP 范围 |
| 不做清单 | 功能、用户群、指标三个维度 | 明确边界、抵抗诱惑 |
| 核心假设 | 明确假设 + 验证标准 | 判断功能是否该做 |


## 可复用的模板清单

本节提供了以下可直接使用的模板：

1. **核心假设模板**：明确你在验证什么
2. **验证标准模板**：定义成功/失败的判断标准
3. **功能筛选表格**：用三个问题快速分类
4. **不做清单模板**：三个维度的完整格式
5. **AI 辅助 Prompt**：让 AI 帮你砍功能和验证判断


## 常见错误 vs 正确做法

| 常见错误 | 正确做法 |
|---------|---------|
| 列了 14 个功能就开始做 | 先明确核心假设，再确定 P0 |
| MVP = 功能最少 | MVP = 投入最小且能验证假设 |
| 做完再给用户看 | 尽早让用户参与验证 |
| 竞品有什么我也做什么 | 只做验证假设必须的功能 |
| 不做清单放脑子里 | 写下来，形成正式的决策记录 |


## 检查清单

在开始动手开发之前，确认以下事项：

- [ ] 我有明确的核心假设（一句话能说清楚）
- [ ] 我有可衡量的验证标准（X 个用户在 Y 时间内做了 Z）
- [ ] 我的 P0 功能不超过 5 个
- [ ] 我写下了「不做清单」
- [ ] 我知道这个 MVP 要验证什么，不验证什么


## 承上启下

回顾一下我们在第二章学到的内容：

- **2.1 思维升级**：从「功能视角」转向「任务视角」，问「用户要完成什么任务」
- **2.2 逆向思维**：先想「什么会导致失败」，提前避坑
- **2.3 减法思维**：用最小投入验证核心假设，学会说「这个不做」

这三个思维工具帮你解决了「做什么」和「不做什么」的问题。

但还有一个问题：**你真的了解你的用户吗？**

下一节，我们将学习「故事思维」——把用户当作故事的主角，深入理解他们的需求和场景。

当你能像讲故事一样描述用户时，你写给 AI 的需求会清晰十倍。
