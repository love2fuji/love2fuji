# 2.6.1 核心洞见：一套思维，多种场景

## 为什么思维工具不是「产品经理专用」

前面五节讲的思维工具，名字听起来都很「产品化」：JTBD、MVP、用户画像、PRD……

但如果你仔细看它们的本质，会发现一个共同点：

**它们都在解决同一个问题——如何把模糊的想法变成清晰的定义。**

| 思维工具 | 表面看是 | 本质是 |
|---------|---------|--------|
| JTBD思维 | 理解用户需求 | 搞清楚「我要完成什么任务」 |
| 逆向思维 | 预防产品失败 | 提前想清楚「什么会出错」 |
| 减法思维 | 砍掉不必要的功能 | 识别「什么是真正重要的」 |
| 故事思维 | 设计用户旅程 | 描述「谁在什么情况下做什么」 |
| 灵魂三问 | 验证产品方向 | 确认「问题定义是否清晰」 |

**「定义问题」这个能力，不管你做什么都需要。**

你做数据分析，需要定义「这个分析要回答什么问题」。
你写自动化脚本，需要定义「这个脚本要解决什么重复劳动」。
你给自己做工具，需要定义「我真正的痛点是什么」。
你给家人做网页，需要定义「他们会在什么场景下使用」。


## 一个贯穿示例：同一个人，四种场景

让我们用一个虚构人物「小王」来说明，同一套思维工具如何应用于不同场景。

**小王的背景**：28岁，在一家电商公司做运营，会用 Excel，不会写代码。最近开始尝试用 AI 编程工具。


### 场景 A：小王要做一份数据分析

**情况**：老板要求小王分析「为什么上个月退货率上升了」。

**没有思维工具时**，小王可能会这样告诉 AI：

```
帮我分析一下这份退货数据
```

AI 会生成一堆图表，但小王不知道该看哪个，也不知道怎么向老板汇报。

**用了思维工具后**，小王会先问自己：

| 问题 | 小王的回答 |
|-----|-----------|
| 谁要看这个分析？（灵魂三问） | 老板，他需要在周会上汇报给CEO |
| 他们要做什么决策？（JTBD思维） | 决定是否调整退货政策 |
| 什么会让这个分析没用？（逆向思维） | 只说「退货率上升了」但不说原因 |
| 最核心的问题是什么？（减法思维） | 退货率上升的主要原因是什么？ |

然后小王会这样告诉 AI：

```
我是电商运营，老板要在周会上向CEO汇报「上月退货率上升原因」。

核心问题：退货率上升的主要原因是什么？

需要的分析：
1. 按商品类别分析，哪类退货最多
2. 按退货原因分析，最常见的3个原因
3. 对比上月和上上月，变化最大的指标

输出要求：3张图表，每张配一句结论，最后给出2-3个可能的原因假设。

数据在这里：[上传Excel]
```


### 场景 B：小王要写一个自动化脚本

**情况**：小王每天早上要从5个平台下载销售数据，手动汇总到一张表里，花费约40分钟。

**没有思维工具时**，小王可能会这样告诉 AI：

```
帮我写一个Python脚本，自动汇总销售数据
```

AI 会问「数据从哪里来」「格式是什么」，小王答不上来，项目就卡住了。

**用了思维工具后**，小王会先评估：

| 问题 | 小王的回答 |
|-----|-----------|
| 这个任务多久做一次？（频率） | 每天 |
| 每次花多少时间？（耗时） | 40分钟 |
| 步骤是否固定？（规则性） | 是，每天都一样 |
| 什么情况下会出错？（逆向思维） | 某个平台改版了、文件格式变了 |
| 先自动化哪一步？（减法思维） | 先做「合并5个Excel」，下载暂时手动 |

然后小王会这样告诉 AI：

```
我的重复任务：每天早上从5个平台下载销售数据Excel，手动合并到一张汇总表。

目前我的工作流程：
1. 手动下载5个Excel（暂时不自动化）
2. 打开每个Excel，复制「日期」「商品名」「销量」「金额」这四列
3. 粘贴到汇总表
4. 按日期排序

希望自动化的部分：
第2-4步，把5个Excel合并成一个汇总表。

文件说明：
- 5个Excel都放在 D:\销售数据\每日\ 文件夹
- 文件名格式：平台名_日期.xlsx（如 淘宝_20241201.xlsx）
- 每个文件的数据在Sheet1，第一行是表头

请帮我写一个Python脚本，实现自动合并。
```


### 场景 C：小王要给自己做一个小工具

**情况**：小王想做一个「每日复盘」小工具，记录每天做了什么、学到什么。

**没有思维工具时**，小王可能会想：

```
我要做一个复盘工具，要有：
- 每日记录
- 周报生成
- 月度统计
- 标签分类
- 搜索功能
- 数据导出
- 图表可视化
……
```

然后这个项目永远做不完。

**用了思维工具后**，小王会先问自己：

| 问题 | 小王的回答 |
|-----|-----------|
| 我真正的痛点是什么？（灵魂三问） | 每天忙完不知道时间花在哪了，第二天又重复同样的错误 |
| 我会在什么场景下用？（故事思维） | 晚上睡前，躺在床上用手机，花2分钟记一下 |
| 什么功能是第一天就必须有的？（减法思维） | 能快速记录「今天做了什么」「明天要改进什么」 |
| 什么会让我做完就不用？（逆向思维） | 太复杂、打开慢、需要填很多东西 |

然后小王会这样告诉 AI：

```
我要做一个极简每日复盘网页。

使用场景：
- 晚上睡前用手机打开
- 2分钟内完成记录
- 不需要登录

核心功能（只需要这些）：
1. 两个输入框：「今天做了什么」「明天要改进什么」
2. 一个保存按钮
3. 能看到最近7天的记录
4. 数据保存在浏览器本地

不需要的功能：
- 不需要标签分类
- 不需要搜索
- 不需要统计图表
- 不需要导出
```


### 场景 D：小王要给爸妈做一个工具

**情况**：小王的妈妈60岁，经常忘记和老朋友视频聊天的时间。

**没有思维工具时**，小王可能会做一个功能很全的「日程管理App」——但妈妈根本不会用。

**用了思维工具后**，小王会先观察和询问：

| 问题 | 小王的回答 |
|-----|-----------|
| 妈妈是谁？技术水平如何？（故事思维） | 60岁，只会用微信，不会装App |
| 她的真实痛点是什么？（灵魂三问） | 不是「管理日程」，而是「记住周三下午3点要和李阿姨视频」 |
| 什么会让她用不了？（逆向思维） | 字太小、步骤太多、需要登录 |
| 她会在什么场景下用？（故事思维） | 周三下午，提前看一眼今天要做什么 |

然后小王会这样告诉 AI：

```
我要给我妈做一个「今日提醒」网页。

用户情况：
- 60岁，只会用微信
- 不会安装App，但会打开微信里的链接
- 视力一般，需要大字

使用场景：
- 我把网页链接发到家庭群
- 妈妈点开就能看到「今天要做什么」

核心功能（只需要这些）：
1. 大字显示「今天是星期X」
2. 显示今天的提醒事项（如「下午3点和李阿姨视频」）
3. 我能远程帮她添加/修改提醒

设计要求：
- 字体至少20px
- 按钮要大，容易点到
- 不需要登录
- 页面打开就能看到信息，不需要任何操作
```


## 共性：都是在「定义问题」

四个场景，表面上完全不同：

- 场景A是数据分析
- 场景B是自动化脚本
- 场景C是个人工具
- 场景D是给家人做工具

但用到的思维工具是完全一样的：

| 步骤 | 思维工具 | 在四个场景中都在问 |
|-----|---------|------------------|
| 第一步 | JTBD + 灵魂三问 | 这个东西要帮谁完成什么任务？ |
| 第二步 | 故事思维 | 在什么场景下使用？ |
| 第三步 | 逆向思维 | 什么情况下会失败/出错/用不了？ |
| 第四步 | 减法思维 | 最核心的功能是什么？先做哪个？ |

**思维工具是「操作系统」，场景只是「应用程序」。**

掌握了思维工具，你就能应对任何场景。
