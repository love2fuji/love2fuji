# 3.7.5 从本章到下一章

## 能力盘点：你现在具备什么

完成第二章和第三章的学习后，你已经具备了以下能力：

### 思维能力（第二章）

| 能力 | 你能做到 |
|-----|---------|
| **问题发现** | 从日常烦恼中识别值得解决的问题 |
| **任务定义** | 用 JTBD 描述用户真正要完成的任务 |
| **风险预判** | 用逆向思维提前识别失败原因 |
| **范围收敛** | 用减法思维确定 MVP 和「不做清单」 |
| **用户理解** | 构建用户画像，描述用户旅程 |
| **自我检验** | 用灵魂三问确认准备就绪 |

### 表达能力（第三章）

| 能力 | 你能做到 |
|-----|---------|
| **上下文构建** | 给 AI 提供完整的项目、任务、个人上下文 |
| **结构化表达** | 用 S.C.A.F.F. 或 R.G.C. 框架组织提示词 |
| **技巧选择** | 根据任务类型选择 Zero-shot/Few-shot/CoT 等技巧 |
| **PRD 编写** | 整合思考成果，写出 AI 能理解的需求文档 |
| **迭代对话** | 通过精确反馈逐步逼近理想结果 |
| **问题处理** | 识别 AI 幻觉，知道何时需要人工干预 |

## 第四章预告：从 0 到 1 开发个人工具

### 你将学到

| 阶段 | 内容 | 你会得到 |
|-----|------|---------|
| **准备** | 整合第二章 + 第三章成果 | 完整的项目 PRD |
| **静态层** | 搭建页面框架和样式 | 可以看的界面 |

| **收尾** | 完整项目回顾 | 你的第一个作品 |

### 贯穿案例

第四章将继续使用小李的待办清单项目。你会看到：

- 如何把第二章的 PRD 转化为第一个 Prompt
- 如何通过迭代对话实现三个 P0 功能
- 如何处理 AI 代码中的 Bug
- 最终得到一个完整可用的待办清单应用

### 你需要准备的

进入第四章之前，请确认：

```markdown
□ 我有一个 AI 编程工具可用（Cursor、Windsurf、Bolt.new 等任意一个）
□ 我完成了第二章的项目思考（或者准备用小李的待办清单跟着做）
□ 我准备好了 2-3 小时的集中学习时间
□ 我有心理准备：AI 可能会犯错，但我知道怎么修正
```


## 课后作业

在进入第四章之前，请完成以下作业：

### 作业一：选择你的项目

从以下选项中选择一个，或者用你自己的想法：

| 项目类型 | 示例 | 难度 |
|---------|------|------|
| **个人工具** | 待办清单、读书笔记、习惯打卡 | ⭐ |
| **数据可视化** | 个人开支分析、健身数据追踪 | ⭐⭐ |
| **自动化脚本** | 文件批量重命名、数据格式转换 | ⭐⭐ |
| **给家人的工具** | 老人用的大字时钟、简单相册 | ⭐ |

**新手建议**：选择「待办清单」跟着教程走完第一遍，然后再做自己的项目。

### 作业二：完成项目 PRD

用本章学到的方法，为你选择的项目完成 PRD。

**PRD 模板**：

```markdown
# [项目名称] PRD

## 1. 项目背景
### 问题来源
[描述你发现的问题]

### JTBD 任务描述
当我 [场景] 时，我想 [任务]，这样我就能 [价值]。

## 2. 目标用户
- 身份：
- 场景：
- 核心需求：

## 3. 功能范围
### P0 功能（第一版必须有）
1. 
2. 
3. 

### 不做清单
- 
- 
- 

## 4. 技术约束
- 技术栈：
- 数据存储：
- 其他：

## 5. 成功标准
[如何验证项目成功]
```

### 作业三：写出第一个 Prompt

基于你的 PRD，用 S.C.A.F.F. 框架写出实现第一个 P0 功能的 Prompt。

**不需要真的发给 AI**，写完保存好，第四章会用到。


## 自我检查清单

在进入第四章之前，确认你能回答这些问题：

### 关于第二章

```markdown
□ 我能用一句话描述项目要解决的问题
□ 我知道目标用户是谁
□ 我明确了第一版要做什么、不做什么
□ 灵魂三问我都能回答
```

### 关于第三章

```markdown
□ 我知道好提示词包含哪些要素
□ 我能选择合适的提示技巧（Zero-shot/Few-shot/CoT）
□ 我知道 AI 输出不满意时如何给反馈
□ 我知道如何验证 AI 推荐的包和 API
```

如果有的问题答不上来，建议回顾对应章节再继续。


## 写在最后

恭喜你完成了第二章和第三章的学习！

这两章是 Vibe Coding 的核心基础。很多人急于动手，跳过思考和表达的训练，结果在开发时反复踩坑、不断返工。

你选择了更扎实的路径：**先想清楚，再说明白，最后才动手**。

这条路看似慢，实际上最快。

下一章，你将看到这些积累如何转化为实实在在的成果——你的第一个 AI 编程作品。


> **记住**
> 
> Vibe Coding 的核心不是「让 AI 写代码」，而是「把你想要的说清楚」。
> 
> 你已经学会了怎么说。现在，让我们开始做。
