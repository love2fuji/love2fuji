# 1.3.3 迷信三："需要懂很多技术名词"

> "什么 API、框架、后端、数据库……听都听不懂，怎么学？"

如果你听到这些词就头大，我理解你。

但我要告诉你一个好消息：**你不需要先学会这些词，再开始做东西。**


## 传统学习 vs AI 时代学习

**传统方式：**
```
学概念 → 学语法 → 学框架 → 做项目
（可能需要半年到一年）
```

**AI 时代的方式：**
```
说出你想要的 → 做出来 → 遇到问题再学
（可能只需要一下午）
```

**顺序完全反过来了。**

你不需要先搞懂什么是"前端"、"后端"、"数据库"，才能做一个网站。

你只需要说："帮我做一个能记录待办事项的网页"。

AI 会帮你处理所有技术细节。


## 那些名词到底是什么？

如果你好奇，我用最简单的话解释几个常见的：

| 技术名词 | 大白话解释 |
|---------|-----------|
| 前端 | 用户能看到的部分（页面、按钮、颜色） |
| 后端 | 用户看不到的部分（数据处理、存储） |
| API | 两个程序之间的"传话筒" |
| 数据库 | 存数据的地方，像个超级 Excel |
| 框架 | 别人写好的代码模板，让你不用从零开始 |
| 部署 | 把你的网站放到网上，让别人能访问 |

**知道这些就够了。** 用到的时候再深入了解。


## 边做边学的正确姿势

与其先啃概念，不如：

### Step 1：先动手做
不管懂不懂，先让 AI 帮你生成代码，看看效果。

### Step 2：遇到问题，问 AI
```
AI 说要配置一个 API，这是什么意思？
能用更简单的话解释一下吗？
```

### Step 3：记住"够用就好"
不需要完全理解每个概念。知道它是干什么的，能用就行。

### Step 4：逐渐积累
做的项目多了，这些词自然就熟悉了。


## 真实案例

**张老师，45 岁，小学语文老师**

她想做一个"学生作业提交"的小网页。

一开始，她完全不懂什么是"前端"、"后端"。

她就直接告诉 AI：

```
我想做一个网页，学生可以输入自己的名字，
然后上传作业文件，我能看到谁提交了。
```

AI 帮她生成了代码。她看了效果，发现可以用。

**三个月后**，她已经知道：
- "前端"就是学生看到的提交页面
- "后端"就是处理上传文件的部分
- "数据库"就是存储提交记录的地方

这些概念，她是**做完项目之后**才真正理解的。

她说：

> "如果让我先学这些概念，我可能学两周就放弃了。但先做出来，再回头看，一下子就懂了。"


## 一个小提醒

虽然不需要先学概念，但有一个能力很重要：

**愿意搜索和提问。**

遇到不懂的词，你可以：
- 直接问 AI："XX 是什么意思？"
- 用搜索引擎查一下
- 在本教程的附录术语表里找

**好奇心比知识储备更重要。**


## 小结

| 迷信 | 真相 |
|-----|------|
| 要先学会技术名词 | 边做边学，遇到再查 |
| 不懂概念就做不了 | AI 会帮你处理技术细节 |
| 学编程要从基础学起 | 先做出东西，再补基础 |

**不要让"不懂"成为"不做"的借口。**


::: tip 给自己的话

就像学游泳。你可以在岸上看一年的教程，也可以直接跳下水扑腾。

哪种方式学得更快？你懂的。

AI 就是你的游泳圈。先跳下去，别怕。
